🔄 DATA FLOW: Trade Submission Example
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

User fills form:
  Symbol: AAPL
  Type: BUY
  Quantity: 50
  Price: 150.25
  Strategy: 1
  
                         ↓
                    
Frontend validates form ✓
  
                         ↓
          
Send JSON to API: POST /api/trades
  
                         ↓
          
TradeController.submitTrade()
  • Validates @NotBlank, @Positive
  • Creates Trade object
  • Calls riskService.evaluateTrade()
  
                         ↓
          
RiskEvaluationService evaluates:
  • ConcentrationRule: 50 × 150.25 = $7,512.50
    → Check: $7,512.50 ≤ 10% of $100,000 = $10,000? ✓ PASS
  • PositionSizeRule: Is 50 between 1 and 10,000? ✓ PASS
  • Result: APPROVED
  
                         ↓
          
If APPROVED (200 OK):
  • Save trade to database
  • Recalculate analytics
  • Return updated portfolio metrics
  
        OR
  
If REJECTED (403 Forbidden):
  • Log RiskFlag violation
  • Return error message
  • Frontend shows alert
  
                         ↓
          
Frontend updates dashboard:
  • P&L card color changes
  • Win rate updates
  • Strategy table refreshes
  • Trade history appends

  • Position Size Limit: 1 - 10,000 shares

TEST TRADES:

❌ REJECTED (Violates Concentration Rule)
   Qty: 100, Price: $150.00
   Value: $15,000 > $10,000 limit
   Error: "Concentration limit exceeded"

❌ REJECTED (Violates Position Size Rule)
   Qty: 10,001, Price: $50.00
   Value: $500,050
   Error: "Position size exceeds maximum"

✅ ACCEPTED (Within limits)
   Qty: 50, Price: $150.00
   Value: $7,500 < $10,000 limit
   Position: 50 shares (1-10,000 range)
   Result: Saved, Analytics updated

✅ ACCEPTED (Mix of trades)
   Multiple trades build portfolio
   P&L calculated: SUM(SELL) - SUM(BUY)
   Win Rate calculated from strategy performance
   Dashboard updated in real-time
